How the systems fit together

cmux tree

What exists right now?

cmux tree --all is authoritative for current windows, workspaces, panes, and surfaces. It cannot answer which Claude session owns a surface.

cmux hook store

Which session claimed it?

Claude lifecycle hooks write session identity and process metadata. One view accumulates history; the other can lag after reattach.

ccs bridge

Is that session live?

ccs reconciles both hook-store views, checks the PID, and intersects the result with the live tree before resume and title decisions.
